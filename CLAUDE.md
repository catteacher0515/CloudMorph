# CLAUDE.md

## 输出规范

- 如果没有特别说明输出格式，默认输出简体中文
- Git Commit 描述使用简体中文
- 在结合代码讲解的时候，如果没有特别需要的情况下，不需要输出导包和package相关的代码

## Bug 沉淀规范

每次成功解决一个 Bug 后，自动触发沉淀机制，输出格式固定为：

- **标题**：简短描述 Bug 的核心问题，便于检索
- **现象**：报错信息或异常表现
- **根源**：为什么会出现这个问题
- **解决方案**：怎么修的
- **如何避免**：下次怎么不踩这个坑

## 学习方式

对于教程中非 CRUD 的内容（定义为"较难内容"），采用以下流程：
1. 由 Claude 主动识别并指出当前较难的内容，输出相关代码
2. 用户用自己的话描述"这段代码是干什么的，我的理解是..."
3. Claude 针对性补充和纠正
4. 用户确认学会后，Claude 输出一个总结版本
5. 不跳过，不死记硬背，强迫输出来暴露真正不懂的地方
